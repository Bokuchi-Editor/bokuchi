# モンキーテストケースリスト

自動テストではカバーしきれないUI統合動作を手動で確認するためのテストケース。
本来はE2Eで実現したいが、今はTauriの対応状況的に不可能なので手動で行う。

## 23. チェックリスト（タスクリスト） — 再発が多い領域

この領域は過去に最低 3 回デグレを起こしています。プレビュー描画 → クリックハンドラ → React state → Monaco モデル同期、と独立した複数のレイヤを経由するため、各レイヤの単体テストが通っても End-to-End では壊れているケースが見抜けません。**リリース前に下記すべて** を実施してください。

| #     | テストケース                                                                          | 操作手順                                                                                                                                                                | 期待結果                                                                                                                                                                                       |
| ----- | ------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| M-127 | プレビューでチェックボックスをクリックするとエディタにも反映される *(再発リグレッション)*       | 分割表示で `- [ ] task` と入力し、プレビュー側に表示されたチェックボックスをクリック                                                                                       | プレビューがチェック済みになり、**エディタのソースも `[ ]` から `[x]` に即座に更新** される。エディタのみ表示に切り替えても変化が反映されており、保存後にディスク上の内容にも反映される         |
