# モンキーテストケースリスト

自動テストではカバーしきれないUI統合動作を手動で確認するためのテストケースです。
本来はE2Eで実現したいが、今はTauriの対応状況的に不可能なので手動で行う。

## 1. タブ操作

| # | テストケース | 操作手順 | 期待結果 |
|---|-------------|---------|---------|
| M-01 | 新規タブ作成 | `Ctrl+N` を押す | 新しいタブが開き、エディタにフォーカスが当たる |
| M-02 | 複数タブ切り替え | 3つ以上タブを開き、各タブをクリック | 正しい内容が表示され、エディタにフォーカスが移る |
| M-03 | Ctrl+Tab タブ切り替え | `Ctrl+Tab` で次タブ、`Ctrl+Shift+Tab` で前タブ | 順序通りに切り替わり、末尾→先頭のラップも動作する |
| M-04 | タブの閉じ（未変更） | 未変更タブの×ボタンをクリック | 確認なしで即座に閉じる |
| M-05 | タブの閉じ（変更あり） | 内容を編集後、×ボタンをクリック | 「保存しますか？」ダイアログが表示される |
| M-06 | 保存ダイアログ - 保存 | M-05のダイアログで「保存」を選択 | ファイルが保存されてタブが閉じる |
| M-07 | 保存ダイアログ - 保存しない | M-05のダイアログで「保存しない」を選択 | 保存せずタブが閉じる |
| M-08 | 保存ダイアログ - キャンセル | M-05のダイアログで「キャンセル」を選択 | ダイアログが閉じ、タブは残る |
| M-09 | 同一ファイルの重複オープン防止 | 同じファイルを2回開く | 2つ目は既存タブにフォーカスが移る（新タブは作られない） |
| M-10 | タブのドラッグ&ドロップ並べ替え | タブをドラッグして順序を変更 | タブの並び順が変わり、永続化される |
