# Unit Test カバレッジ向上計画

## Overview

現在のカバレッジ 24% から効率的に向上させるため、Tauri非依存のコンポーネント・フックから優先的にテストを追加し、段階的にAPI依存のファイルへ拡大する戦略。

## 現状

- **全体**: Stmts 24.24% / Branch 21.43% / Funcs 19.93% / Lines 24.76%
- **テスト済み**: 12 テストファイル / 231 テスト
- **主な未テスト領域**: `src/components/`(大部分 0%）、`src/hooks/`（全て 0%）、`src/api/updaterApi.ts`（0%）

## 戦略

Tauri API への依存度が低い順に攻める。依存度が低いファイルは既存のモック基盤だけでテスト可能で、コスト対効果が高い。
